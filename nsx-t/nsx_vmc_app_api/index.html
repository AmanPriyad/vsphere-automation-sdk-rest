    Operations

    api

    default

    Operation HTTP request Description
    attach_vif POST https://{server}/rest/cloud-service/api/v1/infra/direct-connect/vifs/{vif_id} Perform the attach operation in VMC provider to attach the specified VIF (Virtual Interface) to the SDDC.
    create_public_ip PUT https://{server}/rest/cloud-service/api/v1/infra/public-ips/{public_ip_id} This API is used to create or update a public IP. In creating, the API allocates a new public IP from VMC provider. In updating, only the display name can be modified, the IP is read-only.
    delete_public_ip DELETE https://{server}/rest/cloud-service/api/v1/infra/public-ips/{public_ip_id} Delete a public IP. The IP will be released in VMC provider.
    delete_vif_by_id DELETE https://{server}/rest/cloud-service/api/v1/infra/direct-connect/vifs/{vif_id} Detach and delete a Direct Connect VIF (Virtual Interface) from the SDDC. Detach and delete are one operation in VMC provider, a Direct Connect VIF will be deleted after detached.
    get_dx_bgp_info GET https://{server}/rest/cloud-service/api/v1/infra/direct-connect/bgp Get Direct Connect BGP related information, including current Autonomous System Number of the VGW attached to the VPC
    get_linked_vpc GET https://{server}/rest/cloud-service/api/v1/infra/linked-vpcs/{linked_vpc_id} Get linked VPC information.
    get_mgmt_vm_info GET https://{server}/rest/cloud-service/api/v1/infra/mgmt-vms/{vm_id} Get management VM access information.
    get_public_ip GET https://{server}/rest/cloud-service/api/v1/infra/public-ips/{public_ip_id} Get the public IP information.
    get_sddc_user_config GET https://{server}/rest/cloud-service/api/v1/infra/sddc-user-config Get the user-level SDDC configuration parameters
    list_accounts GET https://{server}/rest/cloud-service/api/v1/infra/accounts Retrieve the shadow account and linked VPC account information from VMC provider. This API is a live query to VMC provider.
    list_advertised_routes GET https://{server}/rest/cloud-service/api/v1/infra/direct-connect/routes/advertised Retrieve BGP routes that are advertised by Direct Connect from VMC provider to on-premise datacenter. This API is a live query to VMC provider.
    list_connected_services GET https://{server}/rest/cloud-service/api/v1/infra/linked-vpcs/{linked_vpc_id}/connected-services List services connected to this linked vpc, for example, S3. The response consist of all available services along with their status.
    list_learned_routes GET https://{server}/rest/cloud-service/api/v1/infra/direct-connect/routes/learned Retrieve BGP routes that are learned by Direct Connect from on-premise datacenter. This API is a live query to VMC provider.
    list_linked_vpcs GET https://{server}/rest/cloud-service/api/v1/infra/linked-vpcs List linked VPC information.
    list_mgmt_vms GET https://{server}/rest/cloud-service/api/v1/infra/mgmt-vms List Management VM information.
    list_public_ips GET https://{server}/rest/cloud-service/api/v1/infra/public-ips List all public IPs obtained in the SDDC.
    list_vifs GET https://{server}/rest/cloud-service/api/v1/infra/direct-connect/vifs Return all non-connected VIFs (with states \"avalible\", \"down\", \"pending\" and \"confirming\") and connected VIFs that are available to the SDDC.
    update_connected_service PUT https://{server}/rest/cloud-service/api/v1/infra/linked-vpcs/{linked_vpc_id}/connected-services/{service_name} Connect/Disconnect the service to the given linked vpc. For example, connect S3. The user will know what services are available through the GET call. If the user is trying to connect/disconnect an unknown service, the POST call will throw a 400 Bad Request error.
    update_dx_bgp_info PUT https://{server}/rest/cloud-service/api/v1/infra/direct-connect/bgp This API is used to create or update DX BGP related information. For ASN update, VIFs should be disconnected from the DX VGW before making this API call. The ASN update operation will be synchronous at this point. In the future the user should make use of the Get RealizationStatus call to check update status. While an ASN update call is in progress, any other DX BGP update request will be rejected.
