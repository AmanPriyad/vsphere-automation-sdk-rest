    api default: update connected service

    Connect/Disconnect the service to the given linked vpc. For example, connect S3. The user will know what services are available through the GET call. If the user is trying to connect/disconnect an unknown service, the POST call will throw a 400 Bad Request error.

    Request:

    HTTP request

    PUT https://{server}/rest/cloud-service/api/v1/infra/linked-vpcs/{linked_vpc_id}/connected-services/{service_name}

    Path Parameters

    Name Type Description
    Required
    linked_vpc_id string linked vpc id (required).
    service_name string connected service name, e.g. s3 (required).

    Request Body Structure:

    {
        "connected_service_status" : {
            "name" : "string",
            "enabled" : true
        }
    }

    Request Body Parameters:

    Name Type Description
    bold = required
    connected_service_status connected_service_status (required).
    connected_service_status.enabled boolean status of service

    Optional. This field may be present in responses from the server, but if it is present in a request to server it will be ignored.
    connected_service_status.name string service name

    Optional. This field may be present in responses from the server, but if it is present in a request to server it will be ignored.

    Response:

    HTTP Status Code: 200

    Representation:

    {
        "value" : {
            "name" : "string",
            "enabled" : true
        }
    }

    value.name string service name

    Optional. This field may be present in responses from the server, but if it is present in a request to server it will be ignored.
